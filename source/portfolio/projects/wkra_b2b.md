#### Overview

I have taken over a very complex backend application, developed initially in PHP in-house web framework,
built on top of a generic ERP system. The whole architecture was one big ball of mud
and resolving the complexity required countless drawings, planning meetings
and post-development features list gathering. Not to mention application
had a huge backlog mostly containing bugs reports, many of which were urgent.

I was working on building more lightweight solution, build with Java and Spring
to encapsulate the 3rd party system complexity and provide a simple API.

Nevertheless is wasn't the complexity that leaded this project to sudden death
- it's was the very poor company health, which unfortunately finished it's history
before releasing the new product.
